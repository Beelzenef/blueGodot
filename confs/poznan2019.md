## Charlas

### _Coding styleguide in Godot_

Para clases:

Para clases, deberíamos seguir el siguiente orden:

* Comentario para nombre de fichero
* Herencia si existe
* Nombre de clase
* `docstring`
* Signals
* Variables `onready`
* `export`s
* `enum`s
* Constantes
* Variables públicas
* Variables privadas
* Código _built-in_
* _Callbacks_ para _signals_
* Otros métodos _built-in_
* Métodos públicos
* Métodos privados

Estructura aconsejable:

```
root
    \_ assets
            \_ art
            \_ fonts
    \_ src
        \_ autoload
        \_ HUD
```
